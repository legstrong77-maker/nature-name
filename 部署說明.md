# 部署說明

## A. 部署網站（讓所有人能用網址打開）

**最簡單：Netlify Drop（30 秒，不用註冊也行）**

1. 打開 https://app.netlify.com/drop
2. 把整個 `nature-name` 資料夾拖進去
3. 它會給你一個網址，例如 `https://lucky-cat-12345.netlify.app`
4. 這個網址就可以分享給所有荒野親子團使用

**進階：GitHub Pages（需要 GitHub 帳號）**

1. 在 GitHub 建立新 repo，把 `index.html` 上傳
2. Settings → Pages → Source 選 main branch → Save
3. 等 1-2 分鐘，會給你 `https://你的帳號.github.io/repo名/` 的網址

## B. 設定 Google Sheet 後端（讓所有人共用同一份資料）

> 這一步是「想讓 A 團團員打開網站，看到管理員匯入的資料」必做的步驟。
> 不做的話只會存在自己瀏覽器，別人看不到。

### 步驟一：建立 Google Sheet

1. 到 https://sheets.google.com 建立新試算表
2. 名稱可以叫「自然名資料庫」之類的

### 步驟二：貼上後端程式碼

1. 在剛剛的 Sheet，上方選單按【擴充功能】→【Apps Script】
2. 把 `apps-script.gs` 整個檔案的內容**全部複製**，貼到打開的編輯器（**蓋掉**原本的 `function myFunction()...`）
3. 上方按💾儲存圖示，專案名稱填「自然名 API」

### 步驟三：部署成 Web App

1. 右上角【部署】→【新增部署作業】
2. 左邊齒輪選擇【網頁應用程式】
3. 設定：
   - **執行身分**：我（你的 Google 帳號）
   - **存取權**：**任何人** ⚠️ 重要！這樣大家才連得到
4. 按【部署】
5. 第一次會跳出授權視窗：
   - 選擇你的 Google 帳號
   - 看到「Google 尚未驗證這個應用程式」**是正常的**（因為這是你自己寫的）
   - 按【進階】→【前往「自然名 API」】→【允許】
6. 部署完成後會給你一段網址，**複製整段網址**：
   ```
   https://script.google.com/macros/s/AKfycb.....長一串...../exec
   ```

### 步驟四：到網站貼上網址

1. 打開你部署好的網站
2. 進入【⚙️ 管理】分頁
3. 最上方的「☁️ 雲端同步設定」框，把剛才複製的網址貼進去
4. 按【連線並同步】
5. 看到上方徽章變成 **☁️ 已同步**，就成功了！

### 步驟五：分享給團員

兩種方式擇一：

- **方式 A**：每個團員自己也照步驟四貼一次網址（同一段 Apps Script 網址，大家共用）
- **方式 B**：管理員按【📋 複製連線網址】，會產生一個**已內建後端網址的連結**，例如：
  ```
  https://lucky-cat-12345.netlify.app/?api=https://script.google.com/macros/s/.../exec
  ```
  把這個連結傳給團員，他們打開就**自動連到雲端**，不用自己設定。

## 常見問題

**Q: 連線後同步失敗 / CORS 錯誤？**
A: 確認部署時「存取權」選的是【任何人】，不是【任何擁有 Google 帳號的人】。

**Q: 修改了 Apps Script 程式碼後，網站怎麼沒反應？**
A: 必須重新部署：【部署】→【管理部署作業】→ 點現有部署 → ✏️ → 版本選【新版本】→【部署】。
   網址不會變，前端不用重設。

**Q: 一個 Sheet 可以給幾個團共用？**
A: 不限。在網站上「＋ 新增團體」可以建立多個團，資料會用 `groupId` 自動分開存在同一個 Sheet。

**Q: 想跟其他組織完全分開，不同 Sheet？**
A: 各自重新做一次「步驟一～三」，每個組織有自己的 Apps Script 網址。

**Q: 資料會不會被別人改掉？**
A: 任何拿到 Apps Script 網址的人都能讀寫。在親子團的信任環境下通常 OK。
   如果需要密碼保護，可以再幫你加（例如管理員操作要密碼，一般使用者只能讀）。

**Q: Google Sheet 免費版額度夠嗎？**
A: Apps Script 每天 20,000 次呼叫，遠超過幾個團的使用量。完全免費。
